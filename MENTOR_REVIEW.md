# Mentor 验收说明

这一版用于项目阶段性验收，重点展示个人博客从 MVP 到可运维系统的完整闭环。

## 已完成能力

- 内容系统：文章发布、编辑、删除、草稿、分类、标签、月份归档、置顶、Markdown、LaTeX、正文图片。
- 互动系统：登录后评论、评论回复、评论审核、点赞、收藏、点踩、问号互动、账号级去重。
- 用户系统：邮箱登录、GitHub OAuth、管理员权限、普通用户账号中心。
- 管理后台：文章管理、评论管理、图片管理、站点统计、AI 写作辅助。
- AI 模块：AI 工作台、AI 候选填入文章、后台润色/续写/大纲、AI 历史持久化、真实模型配置状态和连通性测试。
- 部署运维：Docker Compose、PostgreSQL、Nginx、HTTPS、GitHub Actions 自动部署、备份和运维文档。

## 演示顺序建议

1. 打开首页，说明项目定位和当前 mentor review 阶段。
2. 打开文章页，演示分类筛选、标签筛选、归档、热门文章和文章详情。
3. 登录普通账号或 GitHub，演示点赞、收藏、评论和账号中心。
4. 登录管理员，演示文章发布、正文图片、Markdown/LaTeX 预览、草稿和置顶。
5. 演示评论管理：待审、通过、删除、分页和长文本限宽。
6. 演示站点统计：阅读量、用户、评论、热门文章、分类分布。
7. 演示 AI：工作台生成候选、填入文章、后台润色/续写/大纲、AI 历史。
8. 演示 AI 设置测试：说明 API Key 不入库、不入仓库，只通过环境变量或一次性测试框使用。
9. 打开 GitHub Actions，展示 CI 和 Deploy 成功记录。

## API Key 安全说明

- 不要把 API Key 写入代码、README、截图或 GitHub。
- 服务器长期配置请写入 `.env` 或云服务器环境变量。
- 后台 AI 测试框只用于一次性连通性测试，后端不会把 Key 返回给前端。
- 如果密钥曾出现在聊天、截图或公开页面，应立即到服务商控制台吊销并重新生成。

## 已知外部限制

- 国内和微信内置浏览器访问可能受 ICP 备案和平台策略影响。
- 未配置真实模型时，AI 使用本地回退内容，适合演示流程，不代表最终模型效果。
