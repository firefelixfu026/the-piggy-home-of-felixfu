历史推进记录、旧计划、临时清单和排查记录放在 `docs/archive/` 目录中。

这些文件主要用于追溯项目从 MVP 到正式上线的过程，正常验收和部署时不需要逐个阅读。

当前归档重点：

- `docs/archive/project-history/`：旧需求、代码分析、下一步计划和发布清单。
- `docs/archive/GITHUB_PUSH_TROUBLESHOOTING.md`：早期 GitHub 推送问题排查。
