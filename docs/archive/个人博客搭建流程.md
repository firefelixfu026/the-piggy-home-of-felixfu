# 从0到1搭建实用个人博客流程

本文根据 `教学项目：从0到1做实用的个人博客 - Feishu Docs.pdf` 整理，目标是把“个人博客项目”拆成可以执行的流程，并明确哪些事情需要你亲自完成，哪些事情可以由我辅助完成。

## 1. 项目目标

最终交付物：一个可以通过域名访问的个人博客网站。

最低可接受标准：

1. 搭建起基本的全栈博客框架。
2. 完成基础部署，能够在云服务器上运行。
3. 使用容器化方式打包，方便后续迁移和维护。

核心功能：

- 展示个人介绍、照片、兴趣、经历等信息。
- 发布、上传、存储和展示博客文章。
- 支持评论区。
- 支持登录鉴权。
- 支持用户评论、点赞、收藏等交互。
- 通过域名访问网站。
- 配置 CI/CD，实现代码更新后的自动构建或部署。

进阶功能可选：

- 缓存机制、CDN。
- 数据埋点和访问统计。
- 用户数据分析、用户推荐。
- 文章关键词搜索、语义搜索。
- MCP 接口，让 Agent 后续可以管理博客。
- AI 自动化内容模块，例如每日新闻、GitHub Trending 解读、AI 论文 Digest。
- 带排行榜的小游戏。
- 数字化小人等互动模块。

## 2. 推荐技术路线

PDF 中没有硬性技术栈要求，但推荐使用：

- 前端：React 或 Vue。
- 后端：FastAPI。
- Python 包管理：uv 或 pip。
- 数据库：PostgreSQL，开发早期也可以先用 SQLite。
- 容器化：Docker、Docker Compose。
- 反向代理：Nginx。
- CI/CD：GitHub Actions、GitLab CI 或服务器侧自动部署脚本。
- 部署环境：阿里云、腾讯云、火山引擎等云服务器，或者个人试用服务器。

建议优先选择一个简单、稳定、容易部署的组合：

```text
React + FastAPI + PostgreSQL + Docker Compose + Nginx + GitHub Actions
```

如果你更关注快速完成 MVP，也可以先用：

```text
React + FastAPI + SQLite + Docker Compose
```

等功能稳定后再迁移到 PostgreSQL。

## 3. 总体流程

### 阶段 1：确定需求和内容

你应该做：

- 明确博客主题：技术博客、个人主页、作品集、学习记录，或综合型博客。
- 准备个人资料：头像、照片、简介、兴趣、经历、联系方式。
- 准备第一批文章内容，至少 3 到 5 篇。
- 确定是否需要开放评论、点赞、收藏等用户交互。
- 决定是否需要登录注册功能，以及是否允许访客评论。

我可以辅助完成：

- 帮你把需求整理成项目需求文档。
- 设计博客栏目结构，例如首页、文章列表、文章详情、关于我、归档、搜索、登录页、管理后台。
- 帮你设计数据库表结构和 API 路由。
- 把你的个人资料和文章整理成适合网站展示的 Markdown 或 JSON 数据。

阶段交付：

- 一份明确的功能清单。
- 一份页面结构草图。
- 一份数据结构初稿。

### 阶段 2：建立 Git 和项目仓库

你应该做：

- 创建 GitHub、GitLab 或 Gitee 账号。
- 创建项目仓库。
- 决定仓库是否公开。
- 不要把密码、服务器密钥、数据库密码提交到仓库。

我可以辅助完成：

- 初始化 Git 仓库。
- 编写 `.gitignore`。
- 规划前后端目录结构。
- 编写 README。
- 帮你设置基础分支策略，例如 `main`、`dev`。

阶段交付：

- 一个可提交代码的项目仓库。
- 清晰的项目目录结构。

### 阶段 3：前端页面开发

你应该做：

- 确认页面风格：简洁、作品集风格、技术博客风格、极客风格等。
- 提供头像、照片、站点名称、个人介绍等素材。
- 确认文章展示方式：卡片、列表、时间线、分类标签等。

我可以辅助完成：

- 搭建 React 或 Vue 前端项目。
- 实现首页、文章列表页、文章详情页、关于我页面。
- 实现响应式布局，适配手机和桌面。
- 实现 Markdown 文章渲染。
- 添加评论、点赞、收藏入口。
- 优化 UI、交互和基础可访问性。

阶段交付：

- 可以本地运行的前端页面。
- 能够展示个人信息和文章内容。

### 阶段 4：后端和数据库开发

你应该做：

- 确定是否需要用户注册，还是只保留管理员登录。
- 确定评论是否需要审核。
- 确定文章发布方式：后台编辑、Markdown 文件上传、接口发布。

我可以辅助完成：

- 搭建 FastAPI 后端项目。
- 设计并实现文章、用户、评论、点赞、收藏等数据表。
- 实现文章 CRUD 接口。
- 实现登录鉴权，例如 JWT。
- 实现评论、点赞、收藏接口。
- 编写接口文档。
- 编写基础测试。

阶段交付：

- 可运行的后端服务。
- 可连接的数据库。
- 前端可以通过 API 获取文章和提交评论。

### 阶段 5：容器化和本地一键启动

你应该做：

- 安装 Docker Desktop 或在服务器上安装 Docker。
- 确认本地机器或服务器有足够磁盘空间。
- 保存好数据库账号、密码等环境变量。

我可以辅助完成：

- 编写前端 Dockerfile。
- 编写后端 Dockerfile。
- 编写 `docker-compose.yml`。
- 配置数据库容器。
- 配置环境变量模板，例如 `.env.example`。
- 实现一条命令启动前端、后端、数据库。

阶段交付：

- 可以通过 `docker compose up -d` 启动的完整服务。
- 后端和数据库可以随项目迁移。

### 阶段 6：云服务器部署

你应该做：

- 购买或申请云服务器。
- 完成云平台账号实名认证。
- 选择服务器系统，建议 Ubuntu LTS。
- 配置安全组，开放必要端口，例如 80、443、22。
- 妥善保存服务器 SSH 密钥。

我可以辅助完成：

- 给出服务器初始化命令。
- 编写部署脚本。
- 配置 Docker、Nginx、反向代理。
- 协助排查部署错误。
- 帮你检查端口、防火墙、容器日志。

阶段交付：

- 博客可以通过服务器公网 IP 访问。
- 前端、后端、数据库在服务器上稳定运行。

### 阶段 7：域名、HTTPS 和访问优化

你应该做：

- 购买域名。
- 完成域名实名认证和备案要求，具体取决于服务器地区和云厂商。
- 在域名控制台添加 DNS 解析记录。
- 决定主域名和子域名，例如 `example.com`、`www.example.com`、`api.example.com`。

我可以辅助完成：

- 指导你添加 A 记录或 CNAME 记录。
- 配置 Nginx 域名转发。
- 配置 HTTPS 证书，例如 Let's Encrypt。
- 帮你检查域名解析是否生效。
- 帮你优化静态资源缓存策略。

阶段交付：

- 博客可以通过域名访问。
- 网站支持 HTTPS。

### 阶段 8：CI/CD 自动化

你应该做：

- 确定代码托管平台。
- 在仓库或云平台配置必要的密钥，例如服务器 SSH 私钥、镜像仓库账号。
- 确认哪些分支触发自动部署。

我可以辅助完成：

- 编写 GitHub Actions 或其他 CI/CD 配置。
- 配置自动测试、自动构建 Docker 镜像、自动部署到服务器。
- 配置失败通知或日志输出。
- 帮你排查 CI/CD 权限和密钥问题。

阶段交付：

- 推送代码后自动构建或部署。
- 部署流程可重复、可追踪。

### 阶段 9：进阶功能扩展

你应该做：

- 从进阶功能中选择最想做的 1 到 2 个，不建议一次性全做。
- 明确功能优先级：搜索、统计、AI 自动化、MCP、小游戏等。
- 如果涉及第三方 API，准备账号和密钥。

我可以辅助完成：

- 实现文章关键词搜索。
- 接入语义搜索或向量数据库。
- 增加访问统计和数据埋点。
- 设计用户数据分析面板。
- 实现 GitHub Trending 自动解读、论文 Digest 等 AI 自动化模块。
- 设计 MCP 接口，让 Agent 后续可以管理文章、评论和站点配置。
- 实现带排行榜的小游戏。

阶段交付：

- 至少一个实用或好玩的扩展模块。
- 博客从“能用”升级到“有个人特色”。

## 5. 你和我的分工边界

你必须亲自完成或确认的事情：

- 账号注册、实名认证、购买域名、购买服务器。
- 支付、备案、云厂商控制台里的关键确认操作。
- 提供个人照片、头像、简介、经历、联系方式。
- 决定博客风格、栏目、功能优先级。
- 保存和管理密码、Token、SSH 私钥、数据库密码等敏感信息。
- 最终确认网站内容是否可以公开。

我可以辅助完成的事情：

- 根据你的需求设计技术方案。
- 编写前端、后端、数据库、Docker、Nginx、CI/CD 代码。
- 整理文章内容和 Markdown 格式。
- 生成项目文档、部署文档、接口文档。
- 指导你完成云服务器、域名解析、HTTPS 配置。
- 协助排查报错、日志、部署失败、接口联调问题。
- 帮你逐步实现进阶功能。

我不能替你直接完成的事情：

- 替你购买服务器或域名。
- 替你完成实名认证、备案、支付等需要本人操作的事项。
- 在你没有提供授权的情况下访问你的账号。
- 替你决定哪些个人信息应该公开。

## 6. 建议的首个 MVP 范围

为了 8 周内稳定完成，建议先做以下版本：

1. 首页：展示头像、简介、兴趣、经历、最近文章。
2. 文章列表页：展示所有博客文章，支持分类或标签。
3. 文章详情页：支持 Markdown 渲染。
4. 关于我页面：展示个人信息、照片、联系方式。
5. 管理员登录：只有你可以发布、编辑、删除文章。
6. 评论区：访客可以评论，管理员可以删除不合适评论。
7. Docker Compose：一键启动前端、后端、数据库。
8. 云服务器部署：通过公网 IP 访问。
9. 域名和 HTTPS：通过正式域名访问。

MVP 完成后，再加：

- 点赞、收藏。
- 搜索。
- 访问统计。
- AI 自动化内容。
- MCP 管理接口。

## 7. 开始前的准备清单

你需要准备：

- GitHub、GitLab 或 Gitee 账号。
- 一台可开发的电脑。
- Node.js、Python、Docker 的安装环境。
- 个人头像、照片、简介、经历。
- 至少 3 篇初始博客文章。
- 一个云服务器账号。
- 一个域名，或先用服务器公网 IP 临时访问。

我可以先帮你完成：

- 创建项目目录结构。
- 搭建前端和后端脚手架。
- 写好第一版数据库模型。
- 写好 Docker Compose。
- 写好部署文档。
- 把你的文章整理成网站可读取的格式。

## 8. 推荐执行顺序

建议按下面顺序推进：

1. 先确定 MVP 功能，不急着做所有进阶模块。
2. 先完成静态前端页面，让博客看得见。
3. 再接入后端和数据库，让文章可以被管理。
4. 再做登录、评论、点赞、收藏。
5. 再容器化，保证本地一键启动。
6. 再部署到云服务器。
7. 再绑定域名和 HTTPS。
8. 最后加入搜索、统计、AI 自动化或 MCP 等特色功能。

这样推进的好处是每个阶段都有可见成果，即使中途时间不够，也能保留一个可运行、可展示的博客版本。
