# v2.0.0 Review 清单

本清单用于在 v2.0.0 推送后做一次完整验收。目标不是继续加功能，而是确认博客已经具备可持续维护的基础闭环。

## 基础信息

- 线上地址：`https://www.felixfu.xyz`
- 部署方式：GitHub Actions 推送到阿里云服务器，Docker Compose 运行前端、后端和 PostgreSQL
- 当前阶段：v2.0.0 发布候选

## 必测流程

1. 首页
   - 打开首页，确认阶段指标显示 v2.0.0。
   - 确认首页能力说明包含文章系统、评论互动、图片上传、登录后台和 AI 写作辅助。

2. 文章阅读
   - 打开文章页，确认 Markdown、LaTeX 公式、代码块、表格和图片都能正常显示。
   - 点击文章卡片进入详情页，刷新后不应白屏。

3. 评论互动
   - 未登录时点击点赞、收藏、点踩、问号或评论，应只显示“请先登录”提示，不应直接跳转登录页。
   - 登录后同一账号对同一篇文章的同一种互动只能记录一次。
   - 评论输入超过限制时应被拦截或提示。
   - 评论列表和评论管理都应分页显示，并且长评论不撑破页面。

4. 管理后台
   - 游客侧边栏不显示“管理”入口。
   - 管理员登录后可看到“管理”入口。
   - 文章可发布、编辑、删除，草稿状态不出现在游客文章列表。
   - 评论管理可刷新、分页、删除和审核。

5. 图片上传
   - 封面图上传后能显示预览。
   - 正文图片上传后能插入 Markdown 语法，并在文章预览和详情页显示。
   - 图片管理可列出、复制、插入和删除上传图片。

6. AI 模块
   - AI 工作台三种模式可以生成候选内容。
   - AI 候选可以填入文章表单，并默认作为草稿。
   - 后台正文区域可执行 AI 润色、续写、大纲。
   - “追加 / 插入 / 替换”三种模式都应生效。
   - AI 生成历史出现新记录，最新一条可撤回。

7. 部署和运维
   - GitHub Actions Deploy 成功。
   - 服务器 `docker compose ps` 显示 frontend、backend、postgres 正常运行。
   - `https://www.felixfu.xyz/api/health` 返回健康状态。
   - Certbot dry-run 续期测试通过。

## 已知外部限制

- 国内手机浏览器、微信或 QQ 内置浏览器可能因为 ICP 备案和平台安全策略提示拦截；这是外部访问策略问题，不代表应用代码失败。
- 如果真实 AI Provider 未配置，AI 模块会使用本地回退内容；这保证流程可用，但不是最终真实模型效果。

## Review 后建议进入

- v2.1.0：AI 设置页和使用记录持久化。
- v2.2.0：SEO、站点地图、Open Graph 和分享卡片。
- v2.3.0：访问统计、内容仪表盘和异常监控。
