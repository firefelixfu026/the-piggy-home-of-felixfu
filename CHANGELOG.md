## v2.1.4 - 账号收藏和管理员评论审核开关

- 后端新增 `ADMIN_COMMENTS_REQUIRE_APPROVAL` 环境变量，默认 `false`，管理员评论默认直接公开，也可切换为需要审核。
- 账号活动接口新增 `favoriteArticles`，把收藏文章从普通互动记录中单独整理出来。

## v2.1.3 - 登录注册和 AI 权限收口

- 新增普通邮箱注册流程，普通用户可注册读者账号后评论、点赞、收藏、点踩和使用“？”反馈。
- 管理员初始化改为独立接口 `/api/auth/admin/register`，继续要求 `ADMIN_SETUP_TOKEN`，避免和普通注册混在一起。
- 游客导航收口为文章、游戏和登录，降低无登录状态下的页面复杂度。
- 普通用户不再显示 AI 入口，AI 工作台和 AI 写作辅助只保留给管理员使用。
- README 同步说明普通注册、管理员初始化和 AI 权限边界。

## v2.1.2 - 仓库文档中文化整理

- 根目录 `README.md` 改为中文项目介绍，方便验收时快速了解项目目标、功能、技术栈和部署方式。

## v2.1.1 - AI Codex 中转接口适配

- 后端新增 `AI_API_STYLE` 配置，支持 `openai` 和 `codex` 两种真实模型接口类型。
- Codex 模式会直接请求中转站提供的完整地址，不再自动拼接 `/chat/completions`。
- AI 返回内容解析器兼容 `choices`、`output_text`、`content`、`result`、`output[].content[].text` 和 SSE `data:` 等常见形态。
- 当中转站返回网页 HTML 时，后台会明确提示 Base URL 不是完整 API 地址，避免误判为模型输出。
- 后台 AI 设置和测试面板新增“接口类型”选择，可直接测试 AICodeMirror 等 Codex 中转接口。
- `.env.example`、`docker-compose.yml` 和部署文档补充 Codex 中转配置说明。
